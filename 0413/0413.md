## DB

## 참조

: 참조하는 모델의 외래키는 참조되는 키의 PK를 가리킴



부모테이블: 참조되는 테이블



## Foreign Key 특징

키를 사용하여 부모 테이블의 유일한 값을 참조 -> 참조 무결성



## ForeignKey Field

: 2개의 인자 필요

1. 참조하는 class model (연결할 모델)
2. on_delete



### on_delete 인자

: 외래 키가 참조하는 객체가 사라졌을 때 외래 키를 가진 객체를 어떻게 처리할 지를 정의

(부모 없어지면 어케할거임)

CASCADE : 부모가 사라지면 이를 참조하는 객체도 삭제

 

### 자주 들어오는 질문

Form 

1. 언바운드폼: 데이터가 안들어온 폼
2. 바운드폼: 데이터가 들어온 폼

commit=false = 데이터 베이스에 저장을 하지 않았다.
